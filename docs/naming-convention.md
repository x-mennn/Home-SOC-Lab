# Naming Convention

**Project:** Home SOC Lab

**Status:** Completed

**Version:** 1.0

---

# Objective

This document defines the naming standards used throughout the Home SOC Lab.

A consistent naming convention improves administration, documentation, troubleshooting, and future scalability.

# Folder Naming

The project follows lowercase folder names separated by hyphens.

Examples:

- architecture
- installation
- detection-rules
- incident-reports

---

# File Naming

Documentation files use lowercase names with hyphens.

Examples:

- network-design.md
- vm-specifications.md
- ip-addressing-plan.md

Screenshots follow descriptive names.

Examples:

- vmware-workstation-installed.png
- vmware-virtual-network-editor.png
- vmware-preferences.png

---

# Documentation Standards

Every documentation file includes:

- Project name
- Status
- Version
- Clear section headings
- Consistent formatting

---

# Result

The Home SOC Lab follows a structured naming convention that improves readability, consistency, and long-term maintainability.
