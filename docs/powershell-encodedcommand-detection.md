# PowerShell EncodedCommand Detection

## Objective

Validate the detection of PowerShell executing Base64-encoded commands using Wazuh.

---

## Lab Scenario

Attackers commonly use the `-EncodedCommand` parameter to execute Base64-encoded PowerShell commands. This technique is frequently observed in malware, phishing campaigns, and post-exploitation frameworks to obfuscate malicious commands.

---

## Attack Simulation

Executed the following PowerShell commands on the Windows 10 endpoint:

```powershell
$cmd = "Write-Host Home-SOC-Lab"
$bytes = [System.Text.Encoding]::Unicode.GetBytes($cmd)
$encoded = [Convert]::ToBase64String($bytes)
powershell.exe -EncodedCommand <Base64_String>
```

---

## Detection Validation

- Sysmon Event ID 1 generated successfully.
- Wazuh detected the activity using the built-in rule.
- Rule ID: **92057**
- Detection successfully appeared in the Threat Hunting dashboard.

---

## JSON Investigation

Verified the event JSON to confirm:

- Process Image
- Command Line
- Sysmon Event ID
- Rule Information

---

## Rule Investigation

Built-in Wazuh Rule:

**92057**

Description:

> Powershell.exe spawned a powershell process which executed a base64 encoded command.

The built-in rule successfully detected the simulated activity, so an additional custom rule was not required.
