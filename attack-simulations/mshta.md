# MSHTA Simulation

## Objective

This simulation was performed to validate a custom Wazuh detection rule for `mshta.exe` execution. The generated telemetry was used to verify Sysmon event collection and successful triggering of the custom detection rule.

---

## Technique Overview

MSHTA (`mshta.exe`) is a legitimate Microsoft-signed binary that is commonly abused by attackers to execute HTA applications and scripts. Because it is a trusted Windows binary, it is considered a Living Off the Land Binary (LOLBin) and is frequently associated with defense evasion techniques.

---

## MITRE ATT&CK

| Technique | ID |
|-----------|----|
| Signed Binary Proxy Execution: Mshta | T1218.005 |

---

## Commands Executed

```cmd
mshta.exe vbscript:Execute("CreateObject(""WScript.Shell"").Run ""notepad.exe"":close")
```

> The command was executed on the Windows 10 endpoint to simulate MSHTA execution and generate telemetry for custom rule validation.

---

## Expected Detection

The simulation was expected to generate:

- Sysmon Process Creation (Event ID 1)
- Wazuh alert generated by the custom MSHTA detection rule
- Process information including the executable name, command line, parent process, and user context

---

## Outcome

The simulation successfully generated the expected Sysmon event and triggered the custom Wazuh detection rule, confirming that MSHTA execution was being monitored correctly within the Home SOC Lab.
