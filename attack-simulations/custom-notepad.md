# Custom Notepad Simulation

## Objective

This simulation was performed to validate a custom Wazuh detection rule for `notepad.exe` execution. The generated telemetry was used to verify Sysmon event collection and successful triggering of the custom detection rule.

---

## Technique Overview

Notepad is a legitimate Windows application that was used in this lab as a safe process to validate custom Wazuh rule development. The simulation demonstrates how custom detection rules can be tested using a known process execution.

---

## MITRE ATT&CK

| Technique | ID |
|-----------|----|
| User Execution | T1204 |

> Notepad was executed solely to validate custom detection logic within the Home SOC Lab.

---

## Commands Executed

```cmd
notepad.exe
```

> The command was executed on the Windows 10 endpoint to generate a process creation event for custom rule validation.

---

## Expected Detection

The simulation was expected to generate:

- Sysmon Process Creation (Event ID 1)
- Wazuh alert generated by the custom Notepad detection rule
- Process information including the executable name, command line, parent process, and user context

---

## Outcome

The simulation successfully generated the expected Sysmon event and triggered the custom Wazuh detection rule, confirming that the custom detection logic was functioning correctly within the Home SOC Lab.
